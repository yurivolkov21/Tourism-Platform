# P1.5a — Bookings core (no payment yet) · design spec

**Date:** 2026-06-16 · **Branch:** `feat/p1.5a-bookings-core` · **Phase:** P1.5a
**Refs:** [ADR-0006](../decisions/0006-multi-gateway-momo.md) (Stripe + PayPal) ·
[ADR-0008](../decisions/0008-security-integrity-hardening.md) · donor `modules/bookings` ·
pattern source: P1.4. Depends on P1.4c departures (`DeparturesService` exported).

## Goal

Customer-facing **booking lifecycle** up to (but not including) payment. Port the
donor `modules/bookings` adapting for our schema (EN-only; provider-neutral
`paymentProvider` STRIPE|PAYPAL). **Payment session minting + the authoritative
seat reservation are P1.5b** (Stripe) / **P1.5c** (PayPal) — see "Seat timing".

## Prerequisite — enum migration (ADR-0006 pivot)

`PaymentProvider` enum `MOMO` → `PAYPAL` (no data yet). Prisma migration against
live Supabase (`ALTER TYPE "PaymentProvider" RENAME VALUE 'MOMO' TO 'PAYPAL'`)
folded with the schema edit. After this the API only accepts `STRIPE` | `PAYPAL`.

## In scope (customer surface `/bookings`, JWT-guarded)

- **`POST /bookings`** — create a **PENDING** booking:
  - Resolve `tourSlug` → published tour (404 `TOUR_NOT_FOUND`).
  - Resolve `departureId` under that tour (404 `DEPARTURE_NOT_FOUND`); must be
    `OPEN` (400 `DEPARTURE_NOT_OPEN`) and not strictly past (400 `DEPARTURE_DEPARTED`,
    UTC calendar-date compare; same-day allowed).
  - **Soft** seat-check (best-effort, not a reservation): `seatsTotal - seatsBooked
    >= numAdults + numChildren` else 409 `SEATS_NOT_AVAILABLE`.
  - `totalAmount = (departure.priceOverride ?? tour.basePrice) * totalSeats`;
    `currency = tour.currency`.
  - `paymentProvider` chosen by the buyer (STRIPE|PAYPAL), stored now; the
    checkout session is minted in P1.5b/c.
  - Mint unique `code` (`BK-` + 8 base36; port `booking-code.ts`; retry once on
    collision — DB `@unique` is the backstop).
  - Returns the created booking (no `checkoutUrl` yet).
- **`GET /bookings/me`** — caller's bookings, newest first, top 50 (tour + departure
  joined). 401 `USER_NOT_SYNCED` if `@CurrentUser()` is null.
- **`GET /bookings/:code`** — owner-or-admin; non-owner/missing both → 404
  `BOOKING_NOT_FOUND` (no enumeration leak).
- **`POST /bookings/:code/cancel`** — owner cancels their **PENDING** booking →
  `CANCELLED` (`cancelledAt` set). 409 `BOOKING_NOT_CANCELLABLE` if not PENDING
  (PAID → admin refund, P1.5b). No seat change (none held at PENDING).

## Seat-reservation timing (design — important)

Following the donor: seats are **not** reserved at PENDING. The authoritative
reservation (atomic increment of `seatsBooked`, oversell-guarded) happens at
**payment confirmation** (P1.5b webhook). Rationale: reserving on create forces
an abandoned-cart cleanup job (deferred to pg-boss, P1.x); inbound-tourist volume
is low-contention. The soft-check here only short-circuits an obviously-full
departure before checkout. P1.5b will use an **atomic conditional UPDATE**
(`... SET seats_booked = seats_booked + N WHERE id = ? AND seats_booked + N <=
seats_total`, check affected rows) — pooler-safe, no interactive `FOR UPDATE` tx.

## Out of scope (later sub-phases)

- Checkout session minting + webhooks + seat reservation → **P1.5b** (Stripe), **P1.5c** (PayPal).
- Admin refund (`/admin/bookings/*`) → P1.5b (needs a captured payment).
- Confirmation/refund emails (Resend) + abandoned-cart cleanup → P1.5b / P1.x (pg-boss).

## EN-only adaptations (ADR-0005)

Single `tour.title` (no `titleEn/Vi`); no `Locale`. Otherwise the donor DTO/flow ports directly.

## Trust / integrity

- All `/bookings/*` require a verified Supabase JWT; `userId` from JWT `sub`
  (never the body). Contact fields ARE in the body (booking on behalf of others).
- Owner-or-admin reads collapse to 404 (no code enumeration).
- `numAdults` 1..20, `numChildren` 0..20. `departureId` UUID-validated.
- Money via Prisma `Decimal`; serialized as strings.

## Verification

- **Unit** (`bookings.service.spec.ts`): create resolves tour/departure (404s),
  DEPARTURE_NOT_OPEN / DEPARTURE_DEPARTED, SEATS_NOT_AVAILABLE, totalAmount math
  (override vs base × seats), code uniqueness/retry; `findOwnList` scopes by user;
  `findByCodeForCaller` owner-or-admin else 404; cancel only PENDING.
  Plus `booking-code.spec.ts`: format `^BK-[A-Z0-9]{8}$` over N draws.
- **Gate** (api scope) green.
- **Smoke:** build → kill stale :3000 → boot. `POST /bookings` (no token) → 401;
  `GET /bookings/me` (no token) → 401. (Authed end-to-end create needs a seeded
  tour/departure + JWT → exercised in P1.8 `/seed`.)
