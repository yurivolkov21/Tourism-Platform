# P1.4b — Tours (+ M:N destinations, sub-entities, categories) · design spec

**Date:** 2026-06-16 · **Branch:** `feat/p1.4b-tours` · **Phase:** P1.4b
**Refs:** [data-model.md](../architecture/data-model.md) · ADR-0002 (M:N) · ADR-0005 (EN-only) ·
D-P1.1 (TourPolicy model) · D-P1.3 (compareAtPrice) · D-P1.5 (TourCategory lookup) ·
donor `modules/tours` · pattern source: P1.4a destinations.

## Goal

Public catalog + admin CRUD for **tours**, adapting the donor's `modules/tours`
to our clean schema: **M:N destinations**, `TourCategory` as a **lookup table**,
structured sub-entities (itinerary days, FAQs, policies), and price anchoring.
EN-only. Follows the P1.4a CRUD pattern exactly (public + admin controllers,
`Promise.all` pagination, slugify, `P2002→409` / `P2003→409`, class-validator DTOs).

## In scope

### Tours
- **Public** (`/tours`): `GET /` list (published-only) · `GET /:slug` enriched detail.
  Filters: `categorySlug`, `destinationSlug`, `isFeatured`, `search` (title), page/sort.
- **Admin** (`/admin/tours`, `@Roles(ADMIN)`): list (incl. drafts) · detail · create ·
  patch · delete.
- **M:N** — create/update accept `destinationSlugs: string[]` + `primaryDestinationSlug`;
  service resolves slugs→ids, validates existence + `primary ∈ set`, writes
  `TourDestination` rows (`isPrimary`). Update **replaces** the join set.
- **Category** — create/update reference `categorySlug` (FK `NOT NULL`); validated upfront.
- **Sub-entities (nested replace-all):** `itinerary[]`, `faqs[]`, `policies[]` written
  inside the tour create/update via Prisma **nested writes** (`deleteMany` + `create`) —
  atomic per tour, no explicit `$transaction` (pooler-safe).
- **Arrays** `included/excluded/highlights` = `string[]`; `compareAtPrice` optional on Tour.
- **Publish gate** = `isPublished` (the `isActive` analog); `isFeatured` filter on public list.
- **Detail** = enriched (category, destinations w/ primary, itinerary ordered, FAQs ordered,
  policies ordered). List rows = lean summary (no children).
- **Delete** = two-tier (must be **unpublished** first → 409 `TOUR_IS_PUBLISHED`); FK
  `Restrict` from bookings → 409 `TOUR_HAS_BOOKINGS`. Children cascade.

### Tour categories (minimal lookup CRUD — tours can't be created without one)
- **Public** (`/tour-categories`): `GET /` list active (filter UI).
- **Admin** (`/admin/tour-categories`, `@Roles(ADMIN)`): list · detail · create · patch ·
  delete. Mirrors the destinations CRUD shape (slug/name/description/order/isActive).
  Two-tier delete (deactivate first) + FK `Restrict` from tours → 409 `CATEGORY_HAS_TOURS`.

## Out of scope (later)
- **Departures** → P1.4c (separate sub-phase).
- **Media** (hero/gallery) → P1.6.
- Reviews/wishlist/enquiry surfaces → P1.7. (Their tables exist; no endpoints here.)

## Trust / integrity model
- Public reads force `isPublished: true`; admin sees drafts. Admin routes gated by the
  global `RolesGuard` via `@Roles(ADMIN)`.
- Slug uniqueness = DB constraint (`P2002→409`). Bad `categorySlug` / `destinationSlugs`
  fail **upfront** with `400` (`INVALID_CATEGORY` / `INVALID_DESTINATIONS` listing missing
  slugs / `INVALID_PRIMARY_DESTINATION`) — clearer than a downstream `P2003`.
- Whitelisted `sortBy` (no arbitrary `orderBy`). `pageSize` capped at 100.
- Decimal money fields (`basePrice`, `compareAtPrice`) validated `> 0`; serialize as
  decimal strings (Prisma `Decimal`).

## EN-only adaptations (ADR-0005)
Single `title`/`summary`/`description`/`name`; no `*Vi`. `included/excluded/highlights`
are plain `text[]`. Donor's category **enum** → our `TourCategory` lookup (D-P1.5).
