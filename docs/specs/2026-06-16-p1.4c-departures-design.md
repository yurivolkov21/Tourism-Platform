# P1.4c — Tour departures (nested CRUD) · design spec

**Date:** 2026-06-16 · **Branch:** `feat/p1.4c-departures` · **Phase:** P1.4c
**Refs:** [data-model.md](../architecture/data-model.md) · D-P1.3 (compareAtPrice on Departure) ·
donor `modules/departures` · pattern source: P1.4a/P1.4b.

## Goal

Public read + admin CRUD for **`TourDeparture`** rows nested under a parent tour.
Port the donor `modules/departures` near-verbatim (its date/seat logic is sound)
and add our schema delta: **`compareAtPrice`** on a departure (D-P1.3). EN-only.

## Routes (nested — matches donor + tours resource hierarchy)

- **Public** (`/tours/:slug/departures`, `@Public`): `GET /` — upcoming **OPEN**
  departures for a **published** tour, ordered `startDate` asc.
- **Admin** (`/admin/tours/:slug/departures`, `@Roles(ADMIN)`): `GET /` (full
  history) · `POST /` · `PATCH /:id` · `DELETE /:id`.

**No pagination** — departures are a bounded sub-collection per tour; the donor
returns plain arrays (`TourDeparture[]`) ordered by `startDate` asc. We keep that
(deviates from the P1.4a/b paginated envelope, intentionally).

## Behaviour (ported from donor)

- **Public list** pins `tour.isPublished = true` (404 if draft — same conflation as
  `GET /tours/:slug`), defaults `from = today` (UTC start-of-day) and `status = OPEN`;
  caller may override `from`/`to`/`status`.
- **Admin list** — no `isPublished` gate, no implicit `from`; honours `from`/`to`/
  `status` literally so operators see full history (CLOSED/CANCELLED included).
- **Create** (`tourId` derived from `:slug`, **not** body):
  - 404 `TOUR_NOT_FOUND` if slug missing.
  - 400 `INVALID_DATE_RANGE` if `endDate < startDate`.
  - 400 `DEPARTURE_IN_PAST` if `startDate` (UTC date) < today (same-day allowed).
  - `seatsBooked` is **never** client-settable (defaults 0) — prevents oversell.
  - `priceOverride` / `compareAtPrice` optional Decimal(12,2).
- **Update** (`PATCH /:id`):
  - 404 `TOUR_NOT_FOUND` / `DEPARTURE_NOT_FOUND` (distinct codes).
  - date-range re-validated against existing value when only one date is sent.
  - 400 `SEATS_TOTAL_BELOW_BOOKED` if `seatsTotal < existing.seatsBooked`.
  - `priceOverride` / `compareAtPrice` `null` clears the override (distinguish
    absent vs null); `seatsBooked` not settable.
- **Delete** (`DELETE /:id`):
  - 409 `DEPARTURE_HAS_BOOKINGS` if `seatsBooked > 0` (admins mark CANCELLED instead);
    FK `Restrict` from bookings → same 409 as a race backstop (`P2003`).

## Schema delta vs donor

`compareAtPrice` (Decimal(12,2)?, D-P1.3) on `TourDeparture` — threaded through
create / update (null-clearable) / response DTO. Everything else is 1:1 with donor.

## Out of scope

- Booking-driven `seatsBooked` mutation (P1.5 — bookings own it under tx + row lock).
- Notifications / departure reminders (later).

## Trust / integrity

- Admin routes gated by global `RolesGuard` via `@Roles(ADMIN)`; public `@Public`.
- `:id` validated with `ParseUUIDPipe`. Dates strict ISO-8601. `seatsTotal` 1..1000.
- `DeparturesService` **exported** for the bookings module (P1.5) to read/lock seats.

## Verification

- **Unit** (`departures.service.spec.ts`): create resolves tour / 404 / INVALID_DATE_RANGE /
  DEPARTURE_IN_PAST / seatsBooked-not-settable / compareAtPrice; update SEATS_TOTAL_BELOW_BOOKED +
  partial-date guard + priceOverride-null-clears; remove DEPARTURE_HAS_BOOKINGS + P2003 fallback;
  public list pins isPublished + default from/status; admin list full history.
- **Gate** (api scope) green.
- **Smoke:** build → kill stale :3000 → boot. With no seed data: `GET /tours/:slug/departures`
  on a missing tour → 404 envelope; `GET /admin/tours/:slug/departures` (no token) → 401.

## Deps

None new (Prisma client already has `TourDeparture`; class-validator/swagger in place).
