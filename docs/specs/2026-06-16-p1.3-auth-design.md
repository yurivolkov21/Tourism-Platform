# P1.3 — Auth (Supabase JWT + roles + user mirror) · design spec

**Date:** 2026-06-16 · **Branch:** `feat/p1.3-auth` · **Phase:** P1.3
**Refs:** [backend.md](../architecture/backend.md) · ADR-0008 · donor `common/guards`, `modules/{auth,users}`

## Goal

Authenticate every request with a verified Supabase JWT, enforce roles, and
mirror Supabase users into the local `users` table. Ported from the donor,
EN-only.

## In scope

- **Guards (global, `APP_GUARD`):** `SupabaseJwtGuard` (JWKS verify + HS256
  fallback, alg-confusion whitelist, attaches `req.currentUser`) → `RolesGuard`
  (`@Roles` vs `User.role`).
- **Decorators:** `@Public`, `@Roles`, `@CurrentUser`, `@SupabaseIdentity`;
  `AuthenticatedRequest`/`SupabaseAuthIdentity` types.
- **auth module:** `POST /auth/sync` (CUSTOMER upsert) · `POST /auth/admin/sync`
  (ADMIN, `ADMIN_EMAILS` allowlist). Idempotent; customer sync never demotes.
- **users module:** `GET /users/me` · `PATCH /users/me`.
- Liveness `GET /` marked `@Public`.

## Out of scope (later)

Per-resource authz (P1.4+ controllers add `@Roles(ADMIN)` on admin routes).
Email/2FA flows live in Supabase, not here.

## Trust model

Identity (`sub`, `email`) comes from the **verified JWT only**, never the body.
JWT failures → generic 401 (`code: UNAUTHORIZED`); real reason logged at WARN,
never leaked. Role-less authenticated route = any signed-in user; `@Roles(ADMIN)`
= ADMIN only (403 otherwise).

## EN-only adaptations (ADR-0005)

Dropped the donor's `Locale` enum + the `locale` field from `SyncUserDto` /
`UpdateMeDto` / sync+update logic. `User.locale` keeps its DB default `"en"`.

## Verification

Unit: `RolesGuard` (4 — no-meta/401/403/allow), `AuthService` (3 —
customer-upsert/admin-allowlist-reject/admin-promote). Gate green (21 tests).
**Smoke:** boot OK, DB connected; `GET /api/v1` (public) → 200; `GET /users/me`
(no token) → 401 envelope; `POST /auth/sync` (no token) → 401.

## Deps

`jose` (JWT/JWKS verification).
