# P1.7 — Reviews + Wishlist + Enquiry + Admin-stats (+ user-avatar) · design spec

**Date:** 2026-06-20 · **Phase:** P1.7 · **Sub-phases / branches:**
`feat/p1.7a-reviews` · `feat/p1.7b-wishlist-enquiry` · `feat/p1.7c-stats-avatar`
**Refs:** [ADR-0003](../decisions/0003-enquiry.md) (Enquiry lead) ·
[ADR-0005](../decisions/0005-en-only.md) (EN-only) ·
[ADR-0008](../decisions/0008-security-integrity-hardening.md) (hardening) · donor
`modules/{reviews,wishlist,admin-stats}` (port + EN-only adapt). Tables already in schema
(`Review`, `Wishlist`, `Enquiry` + `EnquiryStatus`); `enquiry` is **net-new** (no donor module).

## Goal

Close out the remaining customer + admin read/write surfaces on top of bookings:
customers **review** paid bookings and curate a **wishlist**; visitors submit **enquiry**
leads; admins **moderate** reviews, work the **enquiry CRM**, read a **dashboard**, and
manage **review/enquiry** rows. Plus wire the **user-avatar** media slot (signable since
P1.6) into `/users/me`. EN-only throughout; donor's `*_en/*_vi` collapse to single columns
and the M:N tour↔destination shape replaces donor's `destinationId`.

## Decisions (confirmed with owner 2026-06-20)

| # | Decision |
| --- | --- |
| D1 | **Ship as 3 sub-phase PRs** (a/b/c) — same cadence as P1.4/P1.5. |
| D2 | **Enquiry = public lead form** with **rate-limit (`@nestjs/throttler`) + honeypot** from day one (ADR-0008 posture for a public DB-writing endpoint). |
| D3 | **User-avatar = dedicated `PUT /users/me/avatar`** (media-set pattern, single asset, `ownerType=USER`/`role=avatar`); `GET /users/me` attaches `avatarUrl`. Not folded into `PATCH /users/me`. |

Standing rules: review only for **PAID** booking, **1 review/booking** (`Review.bookingId UNIQUE`),
caller must **own** the booking; reviews land `isApproved=false`, public list shows approved-only.

## P1.7a — Reviews (`feat/p1.7a-reviews`)

Port `modules/reviews` (service near drop-in; donor already uses `BookingStatus.PAID`,
looks booking up by `code`). EN-only: donor referenced `titleEn` only in admin-stats, reviews
itself is copy-light — no rename needed in the review rows.

- **`ReviewsService`** (port):
  - `createForCustomer(userId, dto)` — look up booking by `code`; `404 BOOKING_NOT_FOUND`;
    `403 BOOKING_FORBIDDEN` if not owner; `400 REVIEW_NOT_ELIGIBLE` unless `PAID`; create with
    denormalised `tourId`; **P2002 → `409 REVIEW_ALREADY_EXISTS`**.
  - `findApprovedForTour(slug, query)` — public, approved-only, **strips PII**
    (`PublicReview`: rating/title/body/createdAt + `reviewer.fullName` only, fallback `Anonymous`);
    `404 TOUR_NOT_FOUND` for unknown/unpublished slug; `Promise.all` rows+count+`_avg(rating)`
    → `meta.averageRating`. (Pooler-safe: no `$transaction`.)
  - `moderateById(reviewId, isApproved)` — admin toggle; `404 REVIEW_NOT_FOUND`.
  - **Admin list** `findAllForAdmin(query)` — paginated, filter `isApproved?`, includes tour slug +
    reviewer name (admin sees PII). `Promise.all` rows+count.
- **Controllers (3, donor split):**
  - `PublicReviewsController` — `GET /tours/:slug/reviews` (`@Public`).
  - `ReviewsController` — `POST /reviews` (authenticated customer; `@CurrentUser`).
  - `AdminReviewsController` — `GET /admin/reviews` · `PATCH /admin/reviews/:id/moderation`
    (`@Roles(ADMIN)`).
- **DTOs:** `CreateReviewDto` (`bookingCode`, `rating` 1–5 int, `title?` ≤120, `body` ≤2000),
  `ListReviewsQueryDto` (page/limit), `ModerateReviewDto` (`isApproved: boolean`),
  `ListAdminReviewsQueryDto` (page/limit/`isApproved?`), response DTOs (`PublicReviewDto`,
  `PaginatedPublicReviewsDto` with `averageRating`, `AdminReviewDto`).
- **Tests:** `reviews.service.spec` — eligibility matrix (no booking / not owner / not PAID /
  happy / duplicate→409); `findApprovedForTour` PII-strip + average; `moderateById` 404 + toggle;
  admin list filter.

## P1.7b — Wishlist + Enquiry (`feat/p1.7b-wishlist-enquiry`)

### Wishlist (port `modules/wishlist`, EN-only adapt)

- **`WishlistService`:** `add(userId, tourId)` upsert (idempotent, **404** on missing/unpublished
  tour); `remove` idempotent `deleteMany`; `findMineWithTour(userId)` newest-first, cap 100, joins
  a **tour preview** + Cloudinary `media[]` via `MediaService.attachToOwners(TOUR, …)`.
  **EN-only adapt:** preview `select` → `{ id, slug, title, summary, basePrice, currency,
  durationDays, isPublished }` (drop donor `titleEn/titleVi/summaryEn/summaryVi/destinationId`;
  M:N now).
- **`WishlistController`** (`@CurrentUser`, customer): `GET /wishlist` · `POST /wishlist`
  (`{ tourId }`) · `DELETE /wishlist/:tourId` (→ 204).
- **DTOs:** `AddWishlistDto` (`tourId` uuid), `WishlistItemDto` (tour preview + `media`).
- Reuses `MediaModule` (one-way, like Tours/Destinations).
- **Tests:** `wishlist.service.spec` — add 404 vs upsert-idempotent; remove idempotent;
  `findMineWithTour` media attach + shape.

### Enquiry (net-new — no donor port)

- **`EnquiryService`:** `create(dto)` — optional `tourId` validated (must exist + published, else
  `404 TOUR_NOT_FOUND`; `null` allowed → general enquiry); insert `status=NEW`.
  `findAllForAdmin(query)` paginated, filter `status?`, newest-first, `Promise.all` rows+count.
  `updateStatus(id, status)` CRM transition; `404 ENQUIRY_NOT_FOUND`.
- **Controllers:**
  - `EnquiryController` — `POST /enquiries` **(`@Public`)** + **`@Throttle`** (e.g. 5/min/IP) +
    **honeypot** (`website` field must be empty → silent `201` drop, no DB write, to starve bots
    of signal). `name/email/message` required, `phone?`, `tourId?`.
  - `AdminEnquiryController` (`@Roles(ADMIN)`) — `GET /admin/enquiries` ·
    `PATCH /admin/enquiries/:id/status` (`{ status: EnquiryStatus }`).
- **DTOs:** `CreateEnquiryDto` (validators + honeypot `website?` empty), `ListEnquiriesQueryDto`
  (page/limit/`status?`), `UpdateEnquiryStatusDto` (`EnquiryStatus`), `EnquiryDto`.
- **Throttler wiring:** add `@nestjs/throttler` (**direct dep** — webpack externalizes; must boot),
  register `ThrottlerModule.forRoot` globally with a generous default and `@Throttle` the public
  enquiry route specifically. `@SkipThrottle` is *not* applied globally — only the enquiry POST is
  tightened; existing routes keep current behaviour.
- **Tests:** `enquiry.service.spec` — create with/without `tourId`, invalid `tourId`→404, status
  update 404 + transition, admin list filter. (Honeypot/throttle asserted at controller/e2e level.)

## P1.7c — Admin-stats + user-avatar (`feat/p1.7c-stats-avatar`)

### Admin-stats (port `modules/admin-stats`, EN-only adapt)

- **`AdminStatsService.getDashboard()`** (port): parallel `groupBy`/`aggregate` +
  one `$queryRaw` monthly bucket (`date_trunc`), all via `Promise.all`. Slices: overview
  (revenue/bookings/paid/conversion/MoM), `bookingsByStatus`, top tours by revenue/rating/wishlist,
  6-month trend. **EN-only adapt:** tour metadata `select` + response field `titleEn` → **`title`**
  (our schema has no `titleEn`). Currency note retained (USD aggregate; multi-currency deferred).
- **`AdminStatsController`** (`@Roles(ADMIN)`): `GET /admin/stats/dashboard`.
- **DTOs:** typed `AdminStatsResponseDto` (Swagger surface for the future admin app).
- **Tests:** `admin-stats.service.spec` — empty DB zeros; bookingsByStatus fill; top-list tour
  metadata join + `<unknown>` fallback; MoM null when <2 months; conversion math.

### User-avatar (wire the deferred P1.6 slot)

- **`UsersService`:** `setAvatar(userId, mediaItem)` — `syncAssets(tx, USER, userId, [item])` in a
  short `$transaction` (single-asset replace-all, `role=avatar`); `getMe` / `updateMe` enrich with
  `avatarUrl` via `MediaService.attachToOwner(USER, user)` (take first `avatar` asset's `url`).
  `DELETE` path = `syncAssets(…, [])` clears it.
- **`UsersController`:** `PUT /users/me/avatar` (`{ media: MediaInputDto }` single) ·
  `DELETE /users/me/avatar`. `GET /users/me` + `PATCH /users/me` now return `avatarUrl` on `UserDto`.
- **Module:** `UsersModule` imports `MediaModule` (one-way). Avatar `MediaInputDto` constrained to
  `type=IMAGE`, `role=avatar` (reject others 400).
- **Tests:** `users.service.spec` (extend) — `setAvatar` replace-all single; `getMe` attaches
  `avatarUrl`; clear → null.

## Out of scope (deferred)

- **Notification emails** on review-approved / enquiry-received → pg-boss outbox (P1.x, ADR-0007).
- **Cloudinary destroy** of replaced avatar objects → reconcile job (P1.x). DB row replaced now.
- **Wishlist pagination** beyond cap-100 (donor's documented edge case) → when a real user hits it.
- **Per-currency revenue** split in admin-stats → when multi-currency tours ship.

## Trust / integrity (ADR-0008)

- **Reviews:** ownership + PAID gate server-side; public list strips `bookingId`/`userId`/email;
  duplicate guarded by unique constraint → 409 (no leak of who reviewed).
- **Enquiry:** public write → throttled + honeypot; all fields class-validated + length-capped;
  `tourId` existence-checked (no dangling FK, schema `onDelete: SetNull` is the backstop).
- **Admin surfaces** (`/admin/reviews`, `/admin/enquiries`, `/admin/stats`) are `@Roles(ADMIN)`.
- **Avatar:** authenticated-self only; `MediaInputDto` restricted to IMAGE/avatar; replace-all in tx.

- **`/gate`** (scope `@tourism/api`): lint + typecheck + test + build green.
