# P1.5b — Stripe · implementation plan

`/gate` (api scope) + smoke before green.

## Tasks

1. **Dep + StripeService** — add `stripe` to `apps/api`. `payments/stripe.service.ts`
   (port): `onModuleInit` builds the client from `stripe.secretKey`; `createCheckoutSession`,
   `constructEvent`, `createRefund`, `client` getter. `toStripeMinorUnits(decimal, currency)`
   helper (zero-decimal set) in `payments/money.ts` + spec.
   - ✅ typecheck; minor-units spec green (USD ×100, JPY/VND ×1).

2. **Checkout** — `BookingsService.startCheckout(code, caller)`: owner-load (404), PENDING
   (409), provider STRIPE (else 400), mint session (metadata bookingId/Code, expiry,
   success/cancel from `FRONTEND_URL`), persist `providerSessionId`, return `{ checkoutUrl,
   bookingCode, status }`; Stripe error → 502 `CHECKOUT_FAILED`. Route `POST
   /bookings/:code/checkout` on `BookingsController` (+ `CheckoutSessionDto`). Inject
   `StripeService` + `ConfigService` (BookingsModule imports PaymentsModule).
   - ✅ spec: 404/409/PROVIDER_NOT_AVAILABLE/persists session/returns url.

3. **Webhook** — `payments/payments.service.ts` (port + adapt): `handleStripeEvent(raw, sig)`
   — verify (400), idempotency via `PaymentEvent {provider:STRIPE,eventId}`, dispatch.
   `onCheckoutCompleted` → **atomic seat-claim CTE** via `$queryRaw` (paid / overbooked→
   `refundOverbookedAndCancel` / already-processed); `onCheckoutExpired` → PENDING→CANCELLED.
   `payments/payments.controller.ts` (`POST /payments/stripe/webhook`, `@Public` +
   `@SkipTransform`, raw Buffer guard). `payments/payments.module.ts` (exports StripeService).
   `main.ts`: mount `express.raw({type:'application/json'})` on the webhook path before
   `setGlobalPrefix`/JSON parser.
   - ✅ spec: bad sig 400, dup skip, completed→paid, overbooked→refund, expired→cancelled,
     unhandled→noop.

4. **Refund** — `BookingsService.refundByAdmin({code, reason, adminUserId})`: PAID +
   `providerPaymentId` guard (400), Stripe refund first (converge on already-refunded),
   then atomic decrement + REFUNDED (gated PAID). `AdminBookingsController` (`@Roles(ADMIN)`,
   `POST /admin/bookings/:code/refund`, `RefundBookingDto`).
   - ✅ spec: state guards, Stripe-first, already-refunded converge, REFUNDED + audit.

5. **Wire + gate** — register `PaymentsModule` + `AdminBookingsController` in `app.module`;
   BookingsModule imports PaymentsModule. `pnpm nx run-many -t lint typecheck test build
   --projects=@tourism/api`. ✅ green.

6. **Smoke** — build → kill stale :3000 → boot: webhook (no sig) → 400; checkout (no token)
   → 401; admin refund (no token) → 401. Routes mapped.

7. **PR** — commit, push, CI green, **pause for review** → rebase-merge + delete branch.
   Update `.remember`.

## Sequencing

1 (Stripe wrapper) → 2 (checkout) → 3 (webhook) → 4 (refund) → 5 (wire/gate) → 6 (smoke) →
7 (PR). 2 & 4 extend BookingsService; 3 is standalone PaymentsService.

## Reused seams

Donor `modules/payments/{stripe.service,payments.controller,payments.service}.ts` +
`bookings.service.refundByAdmin`. Adapt: provider-neutral columns (`providerSessionId`/
`providerPaymentId`, `PaymentEvent.provider`); **atomic CTE seat-claim** instead of FOR
UPDATE; EN-only (`title`, no Locale); **strip email** (→ P1.x); webhook path
`/payments/stripe/webhook`.

## Risks / guards

- Raw-body middleware ordering — if mis-mounted, `constructEvent` fails; the controller
  guards `Buffer.isBuffer(req.body)` → 400.
- `$queryRaw` enum literals + uuid casts — parameterize values; cast `$1::uuid`.
- Refund money path — Stripe call before DB flip; never mark REFUNDED for a refund that
  didn't happen.
