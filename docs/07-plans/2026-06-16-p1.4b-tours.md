# P1.4b — Tours · implementation plan

**Spec:** [../06-specs/2026-06-16-p1.4b-tours-design.md](../06-specs/2026-06-16-p1.4b-tours-design.md)
**Branch:** `feat/p1.4b-tours` · pattern source: P1.4a destinations.

Dependency-ordered. TDD on service logic (slug/validation/pagination/guards). Run `/gate`
(api scope) + smoke before declaring green. Decimal money → validate `>0`, serialize as string.

## Tasks

1. **Tour categories module** (`modules/tour-categories/`) — port the destinations CRUD shape:
   `tour-categories.service.ts` (+ spec), public `tour-categories.controller.ts` (list active),
   admin `admin-tour-categories.controller.ts` (CRUD), DTOs (create/update/list/category/
   paginated), `tour-categories.module.ts` (export service). Two-tier delete + `P2003→409`.
   - ✅ unit tests pass; mirrors destinations semantics (slug VarChar(60)).

2. **Tours DTOs** (`modules/tours/dto/`):
   - `create-tour.dto.ts` — scalars + `categorySlug` (req) + `destinationSlugs[]` (non-empty) +
     `primaryDestinationSlug` (req) + nested `itinerary[]`/`faqs[]`/`policies[]`
     (`@ValidateNested` + `@Type`, child DTOs in `dto/nested/`).
   - `update-tour.dto.ts` — `PartialType(CreateTourDto)`.
   - `list-tours-query.dto.ts` — page/pageSize/search/sortBy(whitelist)/sortOrder +
     `categorySlug`/`destinationSlug`/`isFeatured`/`isPublished`(admin-only).
   - `tour-summary.dto.ts` (lean list row) · `tour.dto.ts` (enriched detail) ·
     `paginated-tours.dto.ts`.
   - ✅ `nx typecheck @tourism/api` green.

3. **ToursService** (`tours.service.ts` + spec) — TDD:
   - `findPublicList` (forces `isPublished:true`, filters) / `findPublicBySlug` (enriched, 404).
   - `findAll` / `findBySlug` (admin, incl. drafts).
   - `create` / `update` — resolve `categorySlug`→id (400 if missing), resolve
     `destinationSlugs`→ids (400 listing missing, 400 if primary ∉ set), slugify, nested
     write of children (`deleteMany`+`create` on update), `TourDestination` rows with
     `isPrimary`. `P2002→409`.
   - `remove` — must be unpublished (409), `P2003→409` (`TOUR_HAS_BOOKINGS`).
   - Pagination via `Promise.all` (count + findMany), whitelisted `orderBy`.
   - ✅ spec covers: slug gen, missing-category 400, missing-destination/primary 400,
     public forces published, pagination meta, P2002→409, delete guards, replace-all shape.

4. **Tours controllers** — public `tours.controller.ts` (`@Public` list/detail) + admin
   `admin-tours.controller.ts` (`@Roles(ADMIN)` CRUD); Swagger tags/decorators per P1.4a.
   - ✅ typecheck green.

5. **Wire modules** — `tours.module.ts` imports `DestinationsModule` + `TourCategoriesModule`
   (reuse their exported services for slug→id validation); register `TourCategoriesModule` +
   `ToursModule` in `app.module.ts`.
   - ✅ app boots; both modules resolve.

6. **Gate** — `pnpm nx run-many -t lint typecheck test build --projects=@tourism/api`.
   - ✅ all green; new tests pass.

7. **Smoke** — build → kill stale :3000 → `node apps/api/dist/main.js`:
   `GET /api/v1/tours` 200 · `/tour-categories` 200 · `/admin/tours` (no token) 401 ·
   `/tours/nope` 404. All enveloped.

8. **Branch + PR** — commit (conventional), push, open PR, **rebase-merge + delete branch**
   (remote `git push origin --delete` + local). Update `.remember` + roadmap (P1.4b ✅).

## Sequencing
1 (categories) → 2 (DTOs) → 3 (service TDD) → 4 (controllers) → 5 (wire) → 6 (gate) →
7 (smoke) → 8 (PR). Categories first because tour create depends on a category existing.
