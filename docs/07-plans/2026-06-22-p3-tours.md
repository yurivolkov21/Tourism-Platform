## Tasks

1. **Filter taxonomy + tags on `TourCardData`** — add `TravelStyle` / `TourTheme` unions + optional
   `travelStyles?` / `themes?` to `tour-card.tsx`; tag every fixture tour (6 styles / 6 themes pool).
   *Acceptance:* `pnpm nx build @tourism/web` green; every fixture tour has ≥1 style + ≥1 theme.

2. **Pure filter logic in `@tourism/core` (TDD)** — `tours-filter.ts`: `durationBucket(days)`,
   `filterTours(tours, f)` (within-OR / across-AND / empty passthrough), `sortTours(tours, sort)`.
   Write `.spec.ts` first (RED), implement (GREEN).
   *Acceptance:* `pnpm nx test @tourism/core` green, ≥80% on new file; cases: single facet, multi-select OR,
   cross-facet AND, no-filter passthrough, empty result, each sort order.

3. **Web tours lib** — `apps/web/src/lib/tours.ts`: `allTours`, `tourSlugs`, `getTourBySlug`,
   `getTourDetail` (placeholder gallery/itinerary/highlights/included/departures, deterministic from duration).
   *Acceptance:* `tourSlugs().length` = total fixture tours; `getTourDetail('ha-long-bay-2d1n')` returns N-day itinerary.

4. **i18n** — `toursPage` + `tourDetail` namespaces (filter labels, sort, empty, booking, itinerary, tag label maps).
   *Acceptance:* i18n builds; no missing-key access in components below.

5. **`ToursFilters` (client)** — facet checkbox groups + Clear all + result count, driven by props
   (selected + onChange). Pure-ish presentational (state owned by parent).
   *Acceptance:* renders 4 facet groups from i18n; toggling calls back.

6. **`ToursListing` (client)** — owns filter state + sort; composes sidebar (`ToursFilters`) on desktop +
   **drawer** (Sheet/Dialog) on mobile + `TourCard` grid + sort `Select` + empty state. Uses `filterTours`/`sortTours`.
   *Acceptance:* selecting a facet narrows the grid; clear-all restores; empty state shows when no match; mobile drawer opens.

7. **`/tours/page.tsx` (static)** — `ContentHero` + `ToursListing` (+ `metadata`).
   *Acceptance:* `/tours` builds **static** (`○`); grid + sidebar render; 0 horizontal overflow @375/768/1440.

8. **Detail blocks** — `BookingBox` (sticky price/departures/Request-to-book→`#contact`), `TourItinerary`
   (Accordion day-by-day), `TourHighlights`, `TourIncluded`. Reuse `Gallery` for photos.
   *Acceptance:* each renders from `getTourDetail`; booking CTA href = `#contact`.

9. **`/tours/[slug]/page.tsx` (SSG)** — `generateStaticParams` from `tourSlugs`, `notFound()` on unknown;
   compose TourHero + 2-col (main blocks + sticky `BookingBox`) + `Gallery` + `PlanTripForm`. `generateMetadata`.
   *Acceptance:* all tour slugs prerender (`●`); unknown → 404; sticky booking works; 0 overflow @375/768/1440.

10. **Wire links** — `TourTile`/`TourCard` → `/tours/<slug>`; navbar `toursMenu` → `/tours` (+ mobile menu).
    *Acceptance:* clicking a tour tile lands on its detail; navbar Tours → `/tours`.

11. **Gate + review** — `/gate` (lint + typecheck + test + build) + no-hex; responsive screenshots both pages;
    update roadmap/frontend docs. **STOP — confirm before merge.**
    *Acceptance:* gate green; docs updated; screenshots captured.
