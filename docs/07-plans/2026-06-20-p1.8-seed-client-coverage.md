## Risks / gotchas

- **New runtime/dev deps must land in the right package:** `openapi-fetch` is a **direct dep of
  `@tourism/core`** (consumed at runtime by FE); `openapi-typescript`/`supertest` are devDeps. Webpack
  externalization rule applies to `@tourism/api` runtime imports only.
- **Seed targets `DIRECT_URL` (5432)** not the pooler — matches the migration engine.
- **e2e + DB in CI:** in-process supertest still needs a DB for real queries → keep e2e OUT of the CI
  gate (local/seed-gated) unless we provision an ephemeral Postgres (deferred).
- **Generator needs the API running** (`nx serve @tourism/api`) before `:api-types`.
- **TS6305 flood / Jest `--testPathPatterns`** gotchas still apply (see handoff).
- Self-signed PAID booking is **seed-only** — do not let it leak into a runtime code path.
