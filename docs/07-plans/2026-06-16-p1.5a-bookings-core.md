# P1.5a — Bookings core · implementation plan

## Tasks

1. **Enum migration** (ADR-0006 pivot) — edit `schema.prisma`: `PaymentProvider`
   `MOMO` → `PAYPAL`; `prisma migrate dev` (rename value) against live Supabase;
   `prisma generate`. ✅ migration applied; client has `PAYPAL`; no other model touched.

2. **booking-code util** — port `booking-code.ts` (`mintBookingCode`: `BK-` + 8 base36)
   + `booking-code.spec.ts` (format regex over ~2000 draws). ✅ spec green.

3. **DTOs** (`modules/bookings/dto/`):
   + `create-booking.dto.ts` — `tourSlug` (kebab), `departureId` (UUID), `numAdults`
     (1..20), `numChildren?` (0..20), `paymentProvider` (`@IsEnum`, STRIPE|PAYPAL),
     contact fields (name/email/phone?/specialRequests?).
   + `booking.dto.ts` — response shape (code, status, totalAmount string, currency,
     provider, contact, tour+departure summary, timestamps).
   + ✅ `nx typecheck @tourism/api` green.

4. **BookingsService** (`bookings.service.ts` + spec) — TDD, port donor minus Stripe:
   + `create(userId, body)` — resolve published tour (404) → departure under tour
     (404) → OPEN (400) → not-past (400) → soft seat-check (409) → amount math →
     unique code → insert PENDING (with `paymentProvider`). Returns booking.
   + `findOwnList(userId)` — own, newest first, top 50, tour+departure joined.
   + `findByCodeForCaller(code, caller)` — owner-or-admin else 404.
   + `cancelOwnPending(code, caller)` — PENDING→CANCELLED (`cancelledAt`); 409 if not PENDING.
   + helpers: `generateUniqueCode`, `assertSeatsAvailable`.
   + ✅ spec covers all guards + amount math + auth-scoping + cancel.

5. **Controller + module + wiring** — `bookings.controller.ts` (`@ApiBearerAuth`,
   `/bookings`, `@CurrentUser`; 401 USER_NOT_SYNCED when null); `bookings.module.ts`
   (export `BookingsService` for P1.5b payment confirm); register in `app.module.ts`.
   ✅ app boots; routes mapped.

6. **Gate** — `pnpm nx run-many -t lint typecheck test build --projects=@tourism/api`. ✅ green.

7. **Smoke** — build → kill stale :3000 → boot: `POST /bookings` (no token) → 401;
   `GET /bookings/me` (no token) → 401. ✅ enveloped.

8. **PR** — commit (conventional), push, PR, CI green, **pause for review** →
   rebase-merge + delete branch. Update `.remember`.
