# P1.1 — Prisma schema + DB infra · implementation plan

**Spec:** [../06-specs/2026-06-15-p1.1-prisma-schema-design.md](../06-specs/2026-06-15-p1.1-prisma-schema-design.md)
**Branch:** `feat/p1.1-prisma-schema` · **Design-first** (migrate deferred, D-P1.6)

Dependency-ordered tasks; each has an acceptance check. TDD applies to pure logic
(little here — mostly schema/config). Run `/gate` (api scope) before declaring green.

## Tasks

1. **Deps** — add to `apps/api`: `prisma`, `@prisma/client`, `@prisma/adapter-pg`, `pg`, `@nestjs/config`, `joi`. (pg-boss/sentry deferred to their sub-phases.)
   - ✅ `pnpm install` clean; versions pinned.
2. **Prisma scaffold** — `apps/api/prisma/schema.prisma` (generator + datasource), `apps/api/prisma.config.ts` (port donor: schema path, migrations dir, seed cmd, `DIRECT_URL`).
   - ✅ `prisma validate` passes on an empty-but-valid schema.
3. **Enums + lookup + core entities** — write all enums; `TourCategory`, `Destination`, `User`.
   - ✅ `prisma validate` + `prisma generate` succeed.
4. **Tour graph** — `Tour`, `TourDestination` (M:N), `TourItineraryDay`, `TourDeparture`, `TourFaq`, `TourPolicy`; relations + indexes per spec.
   - ✅ validate/generate; M:N + indexes present.
5. **Commerce + content** — `Booking` (multi-gateway + `refundedById` SetNull), `Review`, `Wishlist`, `PaymentEvent` (`@@unique([provider,eventId])`), `Enquiry`, `MediaAsset`.
   - ✅ validate/generate; FK onDelete strategies correct.
6. **Initial migration** — `prisma migrate dev --create-only` (no DB apply); then hand-edit the SQL to add: `CREATE EXTENSION citext`, `email citext`, all **CHECK** constraints, **RLS** `ENABLE` per table.
   - ✅ migration SQL contains citext + every CHECK + RLS enable; reviewed.
7. **PrismaService + PrismaModule** — port donor (`PrismaPg` adapter from `DATABASE_URL`, eager `$connect`, `@Global` module, log policy).
   - ✅ `nx typecheck @tourism/api` green; module wired in `app.module` (or stub).
8. **Env validation** — `config/env.validation.ts` (Joi) + namespaced `registerAs` configs; donor set + MoMo + Sentry.
   - ✅ boot with placeholder env validates; missing required key fails fast (unit test on the schema).
9. **Gate** — `pnpm nx run @tourism/api:lint typecheck` + `prisma validate`.
   - ✅ all green; `prisma generate` client present.

## Sequencing

1 → 2 → (3 → 4 → 5 schema build) → 6 (migration) → 7 → 8 → 9. Tasks 7 & 8 are independent of the schema body and can interleave.
