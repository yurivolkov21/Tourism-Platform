# P1.6 — Media (Cloudinary) · design spec

**Date:** 2026-06-17 · **Branch:** `feat/p1.6-media` · **Phase:** P1.6
**Refs:** [ADR-0008](../02-decisions/0008-security-integrity-hardening.md) (MediaAsset =
polymorphic, no hard FK; integrity via reconcile + same-tx cleanup) · donor
`lib/cloudinary-url`, `modules/{uploads,media}`. `MediaAsset` model already in schema.

## Goal

Attach images/video to tours + destinations via **Cloudinary signed direct upload**
(the FE uploads straight to Cloudinary; the backend never proxies file bytes), and
serve delivery URLs derived from the stored `public_id` at read time. Admin-only
writes. EN-only (media carries no copy).

## Flow (signed direct upload)

1. **Sign** — `POST /admin/uploads/signed-url` (ADMIN) → backend validates
   purpose/filename/format, derives folder + sanitized `public_id`, returns a
   Cloudinary upload **signature** envelope (no file proxied).
2. FE uploads the file directly to Cloudinary with those params → Cloudinary returns
   `public_id` + dimensions/format/bytes.
3. **Attach** — admin sends the owner's full desired media set; backend persists
   `MediaAsset` rows (replace-all).
4. **Read** — tour/destination responses include `media[]` with delivery URLs built
   from `public_id` (transforms can change with no data migration).

## Design decision (stated — open to veto)

**Media is managed via a dedicated sub-resource endpoint, NOT folded into tour/
destination create/update.** The donor folded `media[]` into `POST/PATCH /tours`;
our P1.4b deliberately uses Prisma nested writes (no interactive `$transaction`),
and `MediaAsset` is polymorphic (no Prisma relation to nest). A separate
`PUT /admin/{tours|destinations}/:slug/media` is cleaner, keeps the create/update
contract stable, and makes replace-all explicit. Read-attach still enriches the
owner responses.

## In scope

### Shared

- **`libs`/`lib/cloudinary-url.ts`** (port verbatim): `buildCloudinaryUrl(cloudName,
  asset)` → image `f_auto,q_auto`; video raw URL + poster (`posterId` or `so_0` frame).
- Dep: **`cloudinary`** SDK (direct — for `api_sign_request`; webpack externalizes it).

### Uploads (sign)

- `UploadsService.createSignedUploadParams` (port): purpose → resource_type + folder;
  format allowlist (image/video extensions); sanitized timestamped `public_id`;
  HMAC signature via `cloudinary.utils.api_sign_request`.
- `POST /admin/uploads/signed-url` (`@Roles(ADMIN)`) + `CreateSignedUploadUrlDto`
  (`UploadPurpose` enum: TOUR_HERO/GALLERY/VIDEO, DESTINATION_HERO/VIDEO, USER_AVATAR;
  filename regex anti-traversal; optional contentType).

### Media (persist + read)

- `MediaService` (port): `syncAssets(tx, ownerType, ownerId, assets[])` (replace-all),
  `deleteForOwner(tx, ...)`, `attachToOwners(ownerType, owners[])` (batch, no N+1,
  builds URLs). `MediaInputDto` / `MediaItemDto`.
- **Set endpoints:** `PUT /admin/tours/:slug/media` + `PUT /admin/destinations/:slug/media`
  (ADMIN) — resolve slug→id, `syncAssets(TOUR|DESTINATION, id, body.media)` in a short
  `$transaction`. Body `{ media: MediaInputDto[] }` (ArrayMaxSize 30). Returns the set.
- **Read-attach:** tour detail+list and destination detail+list include `media[]`
  (`MediaService.attachToOwners`). Owner services inject `MediaService`
  (Tours/Destinations → MediaModule, one-way).
- **Delete cleanup:** tour/destination hard-delete also `deleteForOwner` (polymorphic
  media has no FK cascade) — in the same `$transaction` as the owner delete.

## Out of scope

- **User avatars** attach/read (the `USER_AVATAR` purpose is signable, but wiring it
  to `/users/me` is a P1.7/account concern).
- **Reconcile job** (orphaned-asset sweep) + Cloudinary destroy on delete → pg-boss
  (ADR-0008 / P1.x). We delete DB rows now; Cloudinary objects are swept later.

## Trust / integrity

- Sign + set endpoints are ADMIN-only. The sign endpoint returns a signature but
  **never proxies bytes**; `api_secret` never leaves the backend.
- Filename regex + format allowlist guard against traversal / wrong resource type.
- `public_id` is server-derived (sanitized + timestamped) — the FE echoes it.
- Replace-all media sync runs in a `$transaction` (admin, low concurrency → pooler-safe).

## Verification

- **Unit:** `cloudinary-url.spec` (image/video/poster URLs); `uploads.service.spec`
  (purpose→folder/resource_type, format-reject 400, public_id sanitize/anti-traversal);
  `media.service.spec` (syncAssets replace-all + sortOrder fallback, attachToOwners URL
  build + grouping, deleteForOwner). Owner services: media attached on read.
- **Gate** (api scope) green.
- **Smoke:** build → boot. `POST /admin/uploads/signed-url` (no token) → 401;
  `PUT /admin/tours/x/media` (no token) → 401; tour/destination detail include `media: []`.
