**Date:** 2026-06-15 · **Branch:** `feat/p1.1-prisma-schema` · **Phase:** P1.1
**Refs:** [BLUEPRINT §6](../BLUEPRINT.md) · [data-model](../01-architecture/data-model.md) · ADR-0002/0003/0005/0006/0007/0008

## Goal

Lay the database foundation for the backend: a clean, EN-only, hardened Prisma
schema + the Prisma 7 wiring (driver adapter, config, env validation). **Design
-first** — schema + migration authored now; `migrate` runs once creds land
(D-P1.6). No domain modules/endpoints in this sub-phase.

## In scope

- `apps/api/prisma/schema.prisma` (all entities below)
- Initial migration SQL incl. **CHECK constraints**, **RLS enable + policies**, `citext`
- `apps/api/prisma.config.ts` (DIRECT_URL migrations) + `PrismaService` (PrismaPg adapter) + `PrismaModule` (@Global) — port from donor
- `apps/api/src/config/env.validation.ts` (Joi) + namespaced configs — extend donor set with **MoMo + Sentry**
- Unit tests for any pure helpers introduced (e.g. nothing heavy yet)

## Out of scope (later sub-phases)

Envelope/guards (P1.2/1.3), CRUD (P1.4), payments/jobs (P1.5/jobs), seed (P1.8).

## Locked decisions baked in

EN-only (ADR-0005) · M:N Tour↔Destination (ADR-0002) · multi-gateway (ADR-0006) ·

## Integrity & security (ADR-0008)

- **FKs:** `Booking.refundedById → User onDelete:SetNull`; Restrict on booking money refs; Cascade on owned children.
- **CHECK (raw SQL in migration):** `rating BETWEEN 1 AND 5` · `seats_booked >= 0` · `seats_booked <= seats_total` · `seats_total >= 0` · `total_amount >= 0` · `num_adults >= 1` · `num_children >= 0`.
- **citext:** enable extension; `User.email citext` unique (DB-level case-insensitive).
- **RLS:** `ENABLE ROW LEVEL SECURITY` on every table (service-role bypasses; backstop). Default deny. Public read policies for `is_published`/`is_active` content deferred until/unless a Supabase-client read path exists.
- **MediaAsset / refundedById:** keep app-level patterns per ADR-0008 + reconcile job (P1.6/jobs).
