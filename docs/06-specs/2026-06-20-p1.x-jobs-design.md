# P1.x — Background jobs: pg-boss outbox + emails + cron · design spec

**Date:** 2026-06-20 · **Phase:** P1.x (last P1 item — reliability layer) · **Sub-phases / branches:**
`feat/p1.x-a-jobs-emails` · `feat/p1.x-b-cron`
**Refs:** [ADR-0007](../02-decisions/0007-pgboss-outbox-jobs.md) (pg-boss outbox + jobs) · donor
`modules/email` (Resend; EN/VI → adapt EN-only — confirm/refund only). RESEND_API_KEY +
RESEND_FROM_EMAIL already in `apps/api/.env`. `pg-boss` + `resend` are **net-new direct deps** of `@tourism/api`.

## Goal

Close the reliability gaps (ADR-0007): guaranteed-ish transactional **emails** (no more
fire-and-forget) and **scheduled cleanup** — all on the existing Postgres via **pg-boss** (no new vendor).

## Decisions (owner-confirmed 2026-06-20)

| # | Decision |
| --- | --- |
| D1 | **Outbox = a DB `outbox` table written atomically with the state change**, drained by a pg-boss worker. Honors ADR-0007 (at-least-once) **without** an interactive tx — fits our atomic-CTE paid path. |
| D2 | **Emails: confirm + refund (port donor, EN-only) + review-approved + enquiry-received (net-new templates).** |
| D3 | **Cron: both** abandoned-`PENDING`-booking cleanup (release seats) **and** media reconcile (orphan sweep + Cloudinary destroy of replaced assets). |
| D4 | **2 sub-phase PRs.** |

**Architecture constraint (load-bearing):** the paid path is `PaymentsService.claimSeatsForPaid` — a
**single-statement `$queryRaw` CTE** (no interactive `$transaction`, pooler-safe). So the outbox row for
a confirmation is written **inside that CTE** (an extra `INSERT ... RETURNING` CTE clause), not via a
wrapping tx. Lower-stakes enqueues (review-approved, enquiry-received) use a short interactive
`$transaction` (admin/low-concurrency) so the state change + outbox row commit together.

## P1.x-a — pg-boss + outbox + emails (`feat/p1.x-a-jobs-emails`)

### Deps

`pg-boss` + `resend` — **direct deps of `@tourism/api`** (webpack externalizes app deps → boot needs them direct).

### Schema (migration)

- **`Outbox` model** (`outbox` table): `id` uuid · `type` (`EmailType` enum: BOOKING_CONFIRMATION,
  BOOKING_REFUNDED, REVIEW_APPROVED, ENQUIRY_RECEIVED) · `payload` Json · `status`
  (`OutboxStatus`: PENDING/SENT/FAILED) · `attempts` Int · `dedupeKey` String @unique (idempotency —
  e.g. `booking-confirmation:BK-…`) · `lastError` String? · `createdAt`/`processedAt`. Index `(status, createdAt)`.
- `prisma migrate deploy` + hand-written enum SQL (Supabase, no shadow DB; DIRECT_URL 5432).

### Email module (port + adapt)

- `EmailService` (Resend; **EN-only** — drop donor `Locale`): `sendBookingConfirmation`,
  `sendBookingRefunded`, **`sendReviewApproved`**, **`sendEnquiryReceived`**. Defensive (never throws to
  caller — the worker records FAILED + retries).
- `email.templates.ts` (EN-only): port `renderBookingConfirmation`/`renderBookingRefunded`; **net-new**
  `renderReviewApproved` + `renderEnquiryReceived`.
- `email.config.ts` (`resendApiKey`, `fromEmail`) + Joi env (keys already present).

### Jobs module (pg-boss)

- `JobsService` (OnModuleInit/OnModuleDestroy): start/stop pg-boss on the Postgres conn; pg-boss owns its
  `pgboss` schema tables. **`drainOutbox` worker** scheduled (e.g. every 30s): selects PENDING `outbox`
  rows oldest-first, dispatches via `EmailService` keyed by `type`, marks SENT or (FAILED + `attempts++`
  - `lastError`) with capped retries/backoff. Idempotent via `dedupeKey`.
- Disable cleanly when `RESEND_API_KEY` absent (log + no-op) so local/dev without email still boots.

### Outbox enqueue points (atomic with the state change)

- **PAID** → `claimSeatsForPaid` CTE also inserts a BOOKING_CONFIRMATION outbox row (same statement).
- **REFUNDED** → admin refund path writes a BOOKING_REFUNDED row in its existing tx.
- **REVIEW_APPROVED** → `ReviewsService.moderateById` (when flipping to approved) wraps update + outbox
  insert in a short `$transaction`.
- **ENQUIRY_RECEIVED** → `EnquiryService.create` wraps create + outbox insert in a short `$transaction`.

### Verification

- Unit: `email.templates.spec` (renders subject/html/text for all 4); `jobs.service.spec` (drainOutbox:
  PENDING→SENT happy, send-fail→FAILED+attempts, dedupe skip, RESEND-absent no-op). Enqueue: service
  specs assert an outbox row is written on each trigger.
- Smoke: boot with pg-boss; confirm worker scheduled; an enqueued row transitions PENDING→SENT (Resend
  sandbox / test key). `/gate` green.

## P1.x-b — Cron jobs (`feat/p1.x-b-cron`)

- **Abandoned-booking cleanup** — pg-boss cron (e.g. */15 min): `PENDING` bookings older than a TTL
  (e.g. 30 min) → mark CANCELLED + **release seats** (decrement `seatsBooked`) atomically. Idempotent.
  Frees the inventory the seat-claim reserved at checkout-start.
- **Media reconcile** — pg-boss cron (e.g. daily): destroy Cloudinary objects whose `MediaAsset` row is
  gone (replaced/deleted). Mechanism: record orphaned `publicId`s (a `MediaGarbage` table, or reuse the
  outbox pattern with a MEDIA_DESTROY type) when `syncAssets`/`deleteForOwner` drops rows, then the cron
  calls Cloudinary `destroy`. Idempotent (destroy is safe to repeat).
- Verification: unit specs (cleanup selects+releases correctly; reconcile destroys orphans only); `/gate`.

## Out of scope (deferred)

- Multi-locale emails (EN-only, ADR-0005). · A dedicated queue (Inngest/Trigger.dev) — only if pg-boss
  limits bite (ADR-0007). · Per-event email preferences / unsubscribe.

## Trust / integrity

- Outbox rows + handlers are **idempotent** (`dedupeKey`; destroy/cleanup safe to repeat).
- EmailService never throws to the request path (a PAID booking must not be undone by a Resend hiccup).
- Cron seat-release + booking-cancel run atomically so seat inventory can't drift.
