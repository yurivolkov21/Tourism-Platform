# P1.5c — PayPal · implementation plan

**Spec:** [../specs/2026-06-17-p1.5c-paypal-design.md](../specs/2026-06-17-p1.5c-paypal-design.md)
**Branch:** `feat/p1.5c-paypal` · builds on P1.5b (same provider-neutral shape).

Security-sensitive (money + webhook). TDD on capture/webhook/refund branches.
`/gate` + smoke before green. Verify exact `@paypal/paypal-server-sdk` v2 method
signatures against the installed types at implementation (read live).

## Tasks

1. **Dep + PayPalService** — add `@paypal/paypal-server-sdk@^2.4.0`. `paypal.service.ts`:
   `onModuleInit` builds the SDK client from `paypal.*` config (mode → sandbox/live).
   `createOrder` → `{ orderId, approveUrl }`; `captureOrder` → `{ captureId, status }`
   (ORDER_ALREADY_CAPTURED = success); `refundCapture` → `{ id, status }`;
   `verifyWebhookSignature(headers, event)` via REST verify endpoint + cached OAuth
   token. Domain return shapes (no SDK type leak). + amount-format spec.
   - ✅ typecheck; unit on amount formatting.

2. **Shared seat-claim** — extract the P1.5b CTE into
   `PaymentsService.claimSeatsForPaid(bookingId, providerPaymentId): 'paid'|'overbooked'|
   'already_processed'`; refactor Stripe `onCheckoutCompleted` to use it. Export
   `PaymentsService` + `PayPalService` from `PaymentsModule`.
   - ✅ existing Stripe webhook tests still green; shared method unit-tested.

3. **Checkout dispatch** — `BookingsService.startCheckout` dispatches on
   `paymentProvider` (STRIPE existing, PAYPAL → `createOrder`, persist `providerSessionId`
   = orderId, return approveUrl). Inject `PayPalService` + `PaymentsService`. Drop the
   `PROVIDER_NOT_AVAILABLE` 400.
   - ✅ spec: PAYPAL path returns approveUrl + persists order id.

4. **Capture endpoint** — `POST /bookings/:code/capture` (JWT owner) →
   `BookingsService.capturePayPal`: owner-load (404), provider PAYPAL (400), already
   PAID → return; capture → `claimSeatsForPaid`; overbook → `refundCapture` + REFUNDED.
   `CaptureResultDto`.
   - ✅ spec: 404/provider/idempotent-PAID/overbook paths.

5. **PayPal webhook** — `PaymentsService.handlePayPalEvent(headers, body)`: verify (400),
   idempotency `PaymentEvent(PAYPAL,eventId)`, dispatch `PAYMENT.CAPTURE.COMPLETED` →
   resolve booking via `custom_id` + capture id → `claimSeatsForPaid`; refunded/denied →
   reconcile/log. `PaymentsController` `POST payments/paypal/webhook` (`@Public` +
   `@SkipTransform`, normal JSON body — no raw mount).
   - ✅ spec: bad-sig 400, dup-skip, capture-completed → claim, unhandled → 200.

6. **Refund dispatch** — `refundByAdmin` dispatches on provider (STRIPE → createRefund,
   PAYPAL → refundCapture); same release-seats + REFUNDED CTE; provider-specific
   already-refunded convergence.
   - ✅ spec: PayPal refund branch + convergence.

7. **Wire + gate** — register routes; `pnpm nx run-many -t lint typecheck test build
   --projects=@tourism/api`. ✅ green.

8. **Smoke** — build → kill stale :3000 → boot: capture (no token) → 401; paypal
   webhook (no/invalid sig) → 400; (with creds) checkout for a PAYPAL booking returns
   an approve URL. Routes mapped.

9. **PR** — commit, push, CI green, **pause for review** → rebase-merge + delete branch.
   Update `.remember`. (Manual end-to-end approve→capture→webhook with ngrok + a sandbox
   buyer is a follow-up once `PAYPAL_WEBHOOK_ID` is set.)

## Sequencing

1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9. Task 2 (shared claim) precedes 4/5 (both use it).

## Reused seams

P1.5b: `StripeService` shape (mirror for PayPal), the atomic seat-claim CTE,
`PaymentEvent` idempotency, `refundByAdmin` structure, `BookingDto`/checkout DTOs.

## Risks / guards

- SDK v2 method shapes — confirm against installed types before wiring (createOrder
  body, capture path to `payments.captures[0].id`).
- Webhook verify is an outbound PayPal call — cache the OAuth token; FAILURE → 400.
- Capture idempotency — handle ORDER_ALREADY_CAPTURED + already-PAID booking.
- Refund convergence differs Stripe vs PayPal — branch the already-refunded check.

## Deferred / follow-ups

- `PAYPAL_WEBHOOK_ID` + live webhook loop (ngrok) — at/after smoke.
- Emails + abandoned-cart sweep → pg-boss outbox (P1.x).
- P1.6 media · P1.7 reviews/wishlist/enquiry · P1.8 seed + shared/core client.
