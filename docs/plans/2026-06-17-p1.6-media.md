# P1.6 — Media (Cloudinary) · implementation plan

**Spec:** [../specs/2026-06-17-p1.6-media-design.md](../specs/2026-06-17-p1.6-media-design.md)
**Branch:** `feat/p1.6-media` · port source: donor `lib/cloudinary-url`, `modules/{uploads,media}`.

TDD on pure logic (URL build, sign params, sync/attach). `/gate` + smoke before green.

## Tasks

1. **cloudinary-url lib** — port `apps/api/src/lib/cloudinary-url.ts` (`buildCloudinaryUrl`)
   + spec (image `f_auto,q_auto`, video URL + poster via `posterId`/`so_0`).
   + ✅ spec green.

2. **Dep + UploadsService** — add `cloudinary`. `modules/uploads/uploads.service.ts`
   (port): `onModuleInit` configures the SDK; `createSignedUploadParams`
   (purpose→resource_type+folder, format allowlist, sanitized timestamped public_id,
   `api_sign_request`). `SignedUploadParams` envelope. `dto/create-signed-upload-url.dto.ts`
   (`UploadPurpose` enum, filename/contentType regex). `admin-uploads.controller.ts`
   (`POST /admin/uploads/signed-url`, `@Roles(ADMIN)`). `uploads.module.ts`.
   + ✅ spec: purpose maps, format-reject 400, public_id sanitize/anti-traversal.

3. **MediaService** — `modules/media/media.service.ts` (port): `syncAssets`,
   `deleteForOwner`, `attachToOwners`/`attachToOwner`. `dto/media.dto.ts`
   (`MediaInputDto`, `MediaItemDto`). `media.module.ts` (export `MediaService`).
   + ✅ spec: replace-all + sortOrder fallback, attach URL build + grouping, delete.

4. **Media-set endpoints** — `SetMediaDto { media: MediaInputDto[] }` (ArrayMaxSize 30,
   ValidateNested). On the admin tours + destinations controllers:
   `PUT /admin/tours/:slug/media` + `PUT /admin/destinations/:slug/media` →
   service resolves slug→id, `$transaction(syncAssets)`, returns the attached set.
   + ✅ typecheck; routes mapped.

5. **Read-attach + delete cleanup** — inject `MediaService` into Tours/Destinations
   services (modules import `MediaModule`). Attach `media[]` on: tour detail+list,
   destination detail+list. On hard-delete, `deleteForOwner` in the owner-delete
   `$transaction`. Extend `TourDetailDto`/`TourSummaryDto`/destination DTOs with `media`.
   + ✅ spec/typecheck; reads include media.

6. **Gate** — `pnpm nx run-many -t lint typecheck test build --projects=@tourism/api`. ✅ green.

7. **Smoke** — build → kill stale :3000 → boot: `POST /admin/uploads/signed-url` (no token)
   → 401; `PUT /admin/tours/x/media` (no token) → 401; a tour/destination detail includes
   `media: []`. Routes mapped.

8. **PR** — commit, push, CI green, **pause for review** → rebase-merge + delete branch.
   Update `.remember`.

## Sequencing

1 (url lib) → 2 (uploads/sign) → 3 (media service) → 4 (set endpoints) → 5 (read-attach +
delete) → 6 (gate) → 7 (smoke) → 8 (PR).

## Reused seams

Donor ports almost verbatim (EN-agnostic). `MediaAsset` schema already present. Adapt:
separate `PUT .../media` endpoints (vs donor's folded `media[]`); `$transaction` wrapper
for the standalone sync.

## Risks / guards

- `cloudinary` must be a **direct** dep (webpack externalizes it) or boot fails — smoke catches.
+ Polymorphic media: delete cleanup must run on owner hard-delete (no FK cascade).
+ Read-attach adds a Tours/Destinations → Media dependency (one-way; no cycle).
+ `$transaction` on the pooler is fine for admin (low concurrency); replace-all is idempotent on retry.
