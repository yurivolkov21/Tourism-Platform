## P1.x-a — pg-boss + outbox + emails · branch `feat/p1.x-a-jobs-emails`

1. Branch from `main`. Deps: `pnpm --filter @tourism/api add pg-boss resend` (direct deps).
2. **Schema:** add `Outbox` model + `EmailType`/`OutboxStatus` enums to `schema.prisma`; write the
   migration (`prisma migrate deploy`, hand-write enum SQL; DIRECT_URL).
3. **Email module:** `email.config.ts` + Joi env; port `EmailService` + `email.templates.ts` EN-only
   (confirm/refund) + net-new `renderReviewApproved`/`renderEnquiryReceived` + the two send methods.
   Test-first `email.templates.spec`.
4. **Jobs module:** `JobsService` (pg-boss start/stop; `drainOutbox` scheduled worker; RESEND-absent
   no-op). Test-first `jobs.service.spec` (drain happy/fail/dedupe/no-op) with pg-boss + Resend mocked.
5. **Enqueue points:** extend `claimSeatsForPaid` CTE (BOOKING_CONFIRMATION); refund path
   (BOOKING_REFUNDED); `ReviewsService.moderateById` approve → `$transaction` + outbox;
   `EnquiryService.create` → `$transaction` + outbox. Extend each service spec to assert the outbox write.
6. Register `EmailModule` + `JobsModule` in `app.module`. `/gate` → smoke (boot, worker scheduled,
   PENDING→SENT via Resend test key) → commit `feat(api): P1.x-a pg-boss outbox + emails …` → PR →
   confirm → merge.

## P1.x-b — Cron · branch `feat/p1.x-b-cron`

1. Branch from `main`.
2. **Abandoned-booking cleanup:** pg-boss cron registration in `JobsService`; handler selects stale
   `PENDING` bookings (TTL) → CANCELLED + seat release (atomic). Test-first `jobs cleanup` spec.
3. **Media reconcile:** record orphaned `publicId`s on `syncAssets`/`deleteForOwner` (a `MediaGarbage`
   table or MEDIA_DESTROY outbox type — decide at build); pg-boss cron calls Cloudinary `destroy`.
   Test-first spec (destroys orphans only; idempotent).
4. `/gate` → smoke (crons scheduled; manual trigger releases seats / destroys a test orphan) →
   commit `feat(api): P1.x-b cron (abandoned-booking cleanup + media reconcile) …` → PR → confirm → merge.

## Risks / gotchas
- **`pg-boss` + `resend` must be DIRECT deps of `@tourism/api`** (webpack externalization) — smoke catches a miss.
- **Outbox-in-CTE:** extending `claimSeatsForPaid`'s raw SQL needs care — keep it one atomic statement;
  add an `INSERT INTO outbox … ON CONFLICT (dedupe_key) DO NOTHING` CTE clause (idempotent on webhook retry).
- **pg-boss owns its `pgboss` schema** — first boot creates tables (needs DB privileges; DIRECT_URL).
- EmailService must **never throw** to the request/webhook path.
- TS6305 flood / `--testPathPatterns` / merge-dance gotchas still apply (see handoff).
